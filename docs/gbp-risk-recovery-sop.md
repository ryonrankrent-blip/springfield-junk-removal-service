# GBP Risk Prevention, Diagnosis & Recovery SOP

**Scope:** Future use only for a genuinely eligible operating business. Springfield's current lead-generation property remains ineligible.

## Prevention Workflow

1. Complete the eligibility checklist in `local-entity-eligibility-sop.md`.
2. Freeze unnecessary business-name, category, address, service-area, phone, website, and hours edits.
3. Audit duplicates, practitioner/department profiles, prior suspensions, and ownership access.
4. Align truthful website, phone, citations, registration, licensing, signage, and operating documents.
5. Prepare ordinary-course evidence before verification; do not manufacture evidence for Google.
6. Log every edit, verification prompt, owner, date, and result.

## Diagnosis

| Symptom | Possible class | Read-only checks |
|---|---|---|
| Profile visible but edits/reviews/features restricted | Soft restriction | Owner access, notifications, policy emails, edit history |
| Profile absent from Maps/Search | Suspension/removal | Status page, emails, duplicate/name search, policy history |
| Verification requested again | Reverifications/data change | Recent edits, ownership, address/category/phone consistency |
| Duplicate warning | Duplicate/merge conflict | Exact name/address/phone/category search; do not create replacement |

## Correct-Before-Appeal Audit

- Business genuinely eligible
- Name follows real-world branding
- Primary/secondary categories reflect actual services
- Address and SAB configuration follow current rules
- No virtual office/mailbox/borrowed address
- Phone, website, citations, registrations, licenses, hours, signage, and documents agree
- Duplicate resolved through legitimate ownership/support path

## Evidence Package

Create an evidence index with file name, source, date, supported fact, privacy classification, and whether Google requested it. Use only truthful current documents. Redact only where permitted; never alter substantive information.

## First Appeal

1. Stop unnecessary edits.
2. Identify and correct the actual issue.
3. Submit one concise appeal through the current official channel.
4. State verified facts, correction made, and evidence provided without speculation or blame.
5. Save confirmation/case ID and wait for the decision.

## Denied Appeal

Review the denial, eligibility, remaining conflicts, evidence quality, and whether a permitted follow-up exists. Do not resubmit repeatedly, create a replacement, change facts to trigger another method, or use evasive narratives.

## Post-Reinstatement Monitoring

For 30 days: minimize edits; monitor public status, ownership, phone, website, hours, categories, reviews, duplicates, and policy messages; log all changes. Then return to normal guarded change control.

## Explicit Prohibitions

Fabricated documents; staged signage; misleading videos; rented addresses used only for Maps; hidden mailbox/virtual-office status; manipulated verification methods; replacement profiles; manufactured/incentivized reviews; deceptive reinstatement claims.

## Incident Log Fields

Profile URL/ID; operator; owner; symptom/date; last known good state; recent edits; suspected policy class; eligibility result; corrections; evidence index; appeal date/case ID; decision; follow-up eligibility; monitoring dates; final status.
